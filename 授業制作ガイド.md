# ゲーム型授業 制作ガイド

児童福祉施設の小4〜6女子クラス向け「画面で体験するゲーム型授業」を、毎回すんなり同じ品質で作るための手引き。既存8作（月よみ堂／ほしずく／ひだまり探偵社／おなか探検隊／星めぐりの塔／みずの旅／ながれる水の探検隊／ちそう発掘隊）から抽出した。

---

## 0. 唯一の品質基準

**「最高」＝2週間後の教室で、その子たちが夢中になり、遊ぶうちに学び、最後に『わかった／もっと知りたい』で終わること。** これだけが基準。子どもの体験に効かない技術的作り込みは削る。

- 学力の幅が大きい前提で「低い床・高い天井」（全員が最初の一歩を踏め、早い子に追加の挑戦）
- 勝ち負けで傷つけない **協力型**
- 教え込まず問いで **予想させる → 操作して確かめる**
- 学びにブランクのある子への配慮：**小4漢字＋ふりがな**、セリフは短文で先生が音読できる

---

## 1. 着手前に必ずやること（選定・相談フェーズ／下位モデル）

1. **画面である必然性を1文で言語化する。** 「紙では見せられない現象を動かす／操作して確かめられる」価値があるか。無ければ、その単元は画面にせずアナログでよいと正直に言う。
   - 必然性が強い型：見えないものを見せる（消化・地層・粒）／時間を操作する（月・星・地層・発芽）／条件を変えて何度も試す（とけ方・流水・発芽）／動く地図で推理（地図）
   - 必然性が弱い型：実物が用意できるもの（電気・磁石）、写真や紙芝居で足りるもの
2. **5つ星ミッションの下書き**（下記2章の型）。
3. **世界観・案内役**の方向（新規 or 既存キャラ）。
4. **紙との対応装置**をどうするか（下記4章）。
5. **数値・用語の出典**を決める（本物であること。5章）。
6. 合意した設計方針を **memory の project ファイルに書き残す**（実装フェーズのFableがゼロから考えず着手できる）。

---

## 2. 授業の流れ＝5つ星構成（全作共通）

`FLOW` 配列で管理。各ステップは `STEPS{ enter(), tick?() }`。

1. **intro** — 世界観の導入。案内役が「今日はこれを一緒に確かめよう」。
2. **vote（よそうタイム）** — 誤概念を表に出す3択。**答えはここで言わない**（最後に種明かし）。シートにも〇。
3. **ミッション×4** — 毎回このリズム：**①シート/ボタンで予想 → ②代表が操作 → ③判定 → ④たいせつポイント＋バッジ点灯**。難易度は階段状（易→難、または知識を積み上げて最後は応用推理）。
4. **はかせチャレンジ** — 発展・総合。**防災や生活・命の話**に着地できるとよい（流水＝川に近づかない、地層＝がけに近づかない、水＝節水、消化＝よく噛む）。
5. **ending** — 穴埋めまとめ（タッチで答え）＋ **よそうタイムの種明かし** ＋ **おうちミッション**（今日の学びを家/自分の体/近所で再現）。
6. **free（じゆうモード）** — 早い子・時間が余ったとき。子ども同士でクイズを出し合う。

補助機能（全作共通で入れる）：
- **URLに `?step=m_xxx`** で途中から開始（授業中断時の復帰）。
- **せんせい用バー**（右下・半透明）：🔊音 / ◀まえ / つぎ▶（スキップ）/ ↺リセット。
- 45分が押したら「はかせチャレンジ」をスキップ可（**まとめは削らない**）と先生用メモに明記。

## 3. 画面の骨格（全作共通レイアウト）

```
┌─ header：ロゴ ＋ バッジ★×5（達成で点灯） ────────────┐
├─ main ────────────────────────────────────┤
│  [ 左 = 大きなステージ canvas ]   [ 右上 = 丸窓 canvas ]  │
│    体験の主舞台                    見えないものビューア   │
│    ・操作ボタン列(下端)            /別視点/むしめがね     │
│                                  [ 状態ラベル ]         │
│                                  [ ミッションカード ]    │
│                                   案内役キャラ＋吹き出し  │
│                                   おねがいカード         │
│                                   選択肢/進行ボタン列     │
└─ せんせい用バー(右下すみ・半透明) ／ オーバーレイ(導入・まとめ) ┘
```

- **左の大ステージ**＝操作して現象が起きる場所。
- **右上の丸窓**＝単元ごとに役割が変わる「もう一つの目」：そらの窓（月）／星のむしめがね（星）／まほうのめがね・むしめがね（とけ方・消化・水・流水・地層）／ほういじしん（地図）。**見えないものを見せる or 別視点**。
- テーマ色は単元で固定（夜＝紺、体内＝暖色、水＝水色、地層＝土色、地図＝紙…）。明/暗は単元で決め打ち（教室のプロジェクタは明色が見やすいことも多い）。

## 4. 紙との対応装置（設計の肝）

画面の予想を **番号・記号で「言える／書ける」** 仕組みを必ず入れる。これが協力型の全員参加を生み、かつ **それ自体が本物のスキル** になっている。

- 月：軌道上の位置①〜⑧　／　星：北極星まわりの①〜⑧
- 地図：あ〜お×1〜4の座標（＝地図帳のさくいん）／八方位
- 消化・水：口→…→の5ステーション　／　地層：層①〜⑥（＝層序）

シートの図と画面の配置を**完全一致**させる（子どもが「⑤だと思う！」と番号で予想を書ける）。

---

## 5. 品質の原則：数値・記号・用語は本物

**ウソの数値は学びが転移しないので使わない。** 実例：

- とけ方：食塩・ミョウバンの溶解度は理科年表の実測値（20℃/60℃）
- 消化：消化管 約9m、1人1日の水 約200L、歯みがき30秒で約6L
- 星：星は1時間に約15度＝3時間で45度、北極星は2等星
- 地図記号：国土地理院準拠（**病院は⊕でなく十字＋脚。⊕は保健所**——過去に間違えた）
- 流水/地層：上流ほど角ばり大きい石、下流/下層ほど…、地層累重の法則（下ほど古い）、火山灰の粒は角ばる

材料名は世界観に寄せてよいが、画面内で正体を明言する（「ほしの塩＝食塩」「つきのミョウバン＝ミョウバン」）。

## 6. 技術パターン（コード資産）

新規は **既存作をコピーして中身を差し替える** のが最速。骨格が最も洗練されているのは **「ながれる水の探検隊」または「ちそう発掘隊」**（横長対応・2ビュー・粒可視化・時間送りが揃っている）。コピー元候補：
- 操作ボタン中心（ドラッグなし）：ほしずく／おなか／みずの旅／ながれる水／ちそう
- ドラッグ/タッチ操作：月よみ堂／ひだまり探偵社

流用できる共通コード：
- **音**：WebAudioで外部ファイルなし。`tone()` ＋ 効果音（sTap/sYes/sHmm/sFanfare…）。`ac()` でユーザー操作時に AudioContext を resume。
- **canvas**：`fitCanvas()`（dpr対応）／`mapTransform()` は **上部に `PAD_T` のヘッダー帯を予約**（横長モニターで座標ラベルとタイトルが衝突しないための必須対策）。
- **時間送りアニメ**：`S.anim` に from/to/dur を持たせてループで補間。
- **粒の可視化**：右上丸窓に粒を描く（scopeMode で切替）。ラベルは半透明背景を敷いて円のふちで切れないように。
- **キラキラ**：`burst()` / `drawFx()`（星形パーティクル）。
- **CSS**：`clamp()` と vh/vw でレスポンシブ。選択肢ボタンは `flex-wrap`。

## 7. 成果物セット（この形に限定）

1. **じゅぎょう画面 HTML**（1ファイル・オフライン・ブラウザで開くだけ。Next.js/ビルド/外部依存なし）
2. **A4記録シート HTML**（1ページ目＝子ども用/人数分、2ページ目＝先生用進行メモ）。白背景でインクにやさしく。PDF出力で **ちょうど2ページ** に収める。

先生用メモに必ず入れる：準備チェック／45分の流れ表／答え／数値のたね明かし／声かけ・安全／発展。

---

## 8. 制作フローとモデルの使い分け

- **選定・相談・設計方針の言語化 → 下位モデル**（Opus等）。memory に残す。
- **実装 → Fable（最上位・固定）**。品質が体験を左右する（記号・数値・レイアウト・誤概念の判断が実装に埋まっている）。
- フェーズとモデルのミスマッチに気づいたら一言添える（例：実装なのに下位なら「Fable推奨」）。

## 9. 実装後チェックリスト（QA）

- [ ] `node --check` でJS構文OK（HTMLから`<script>`を抜いて検査）
- [ ] Edgeヘッドレスで全場面スクショ確認（`--headless --screenshot --virtual-time-budget=4000`）
- [ ] **横長（1920×950）でも**タイトル/チップ/座標ラベルが重ならない、はみ出さない
- [ ] 選択肢ボタンが画面外に見切れない（3択が縦に収まる）
- [ ] 丸窓スコープのラベルが円のふちで切れない
- [ ] A4シートはPDF出力で2ページに収まる（`--print-to-pdf`、`/Count`でページ数確認）
- [ ] 単元の数値ロジックを node で単体検証（判定条件・角度・溶解度・方位など）
- [ ] せんせいバー（前/次/リセット/音）と `?step=` 復帰が動く
- [ ] スキル：スクショQAや数値検証は Node/Edge スクリプトに寄せ、思考を判断に集中させる
